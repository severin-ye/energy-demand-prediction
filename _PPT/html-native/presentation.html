为什么并行优于串联？

串联架构 (S-CNN-LSTM)

Input (20, features)
      ↓
  Conv1D(64)
      ↓
  MaxPool(2) → 压缩至(10, 64)
      ↓
  Conv1D(128)
      ↓
  MaxPool(2) → 压缩至(5, 128)
      ↓
  【信息瓶颈】
      ↓
  LSTM(128) → 只能看压缩后的特征
      ↓
  Output

问题：

  • CNN降维 → 信息损失
  • LSTM只能处理压缩后的数据
  • 无法利用原始时序信息
VS

并行架构 (P-CNN-LSTM)

     Input (20, features)
      ┌─────┴─────┐
      ↓           ↓
  CNN分支      LSTM分支
   降维到       保留原始
   空间特征     时序信息
      ↓           ↓
  Flatten(64)  Context(128)
      └─────┬─────┘
            ↓
    【特征丰富】192维
            ↓
         Output

优势：

  • 保留更多原始信息
  • 特征维度更高(192 vs 128)
  • 性能提升34.84%

注意力机制详解

公式推导

et = tanh(We · ht + be)

αt = exp(et) / Σj exp(ej)

c = Σt αt · ht

符号说明

et 时间步t的重要性得分

c 加权上下文向量
We, be 可学习的权重和偏置

状态分类器 - 三状态定义

分类公式

Sn = {

Peak, if ŷt+1 > μ + σ

Normal, if μ-σ ≤ ŷt+1 ≤ μ+σ

Lower, if ŷt+1 < μ - σ

}

符号 含义
ŷt+1 预测的下一时刻能耗
μ 观测窗口内能耗均值
σ 观测窗口内能耗标准差

状态意义

🔴 Peak (峰值)

高能耗状态，需要节能建议
例：开空调+洗衣机+做饭

🔵 Normal (正常)

稳定状态，维持现状
例：日常照明+电视

🟢 Lower (节能)

节能成功，可维持
例：关闭大功率电器

因果推断与推荐生成

Pearl因果框架

观察 (Seeing):

P(Y | X=x) - 被动观测

干预 (Doing):

P(Y | do(X=x)) - 主动控制

P(State=Lower | do(Temp=Low))

推荐流程

  1. 用户当前数据 → 预测状态
  2. 如果State=Peak → 寻找干预
  3. 枚举可控变量组合
  4. do-calculus推断
  5. 选择P(Lower)最大的干预
  6. 生成自然语言建议

推荐示例

场景：预测State=Peak

干预方案 P(Lower) 可行性
do(Temp=Low) 0.65 ✅ 降低空调
do(Humidity=Dry) 0.42 ❌ 难控制
do(Time=Night) 0.78 ⏰ 延迟用电

预测性能指标

MAE (Mean Absolute Error)

MAE = (1/n) Σ |yi - ŷi|

RMSE (Root Mean Squared Error)

RMSE = √[(1/n) Σ (yi - ŷi)²]

MSE (Mean Squared Error)

MSE = (1/n) Σ (yi - ŷi)²

解释稳定性指标

余弦相似度

Sim(e₁, e₂) = (e₁ · e₂) / (||e₁|| · ||e₂||)

用途：衡量多次解释的一致性

现象：训练集95%, 测试集72%
原因：候选边过多
解决：提高min_confidence (0.5→0.7)

问题4: 推荐生成不稳定

现象：部分推荐不可行
原因：未过滤不可控变量
解决：添加可控变量白名单
