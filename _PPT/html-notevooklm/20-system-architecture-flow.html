闭环执行与反馈路径

🧩 六大模块协同（文字概览）

  • Context Encoder：抽取季节、建筑形态、实时负荷与模型内部状态，生成稳健上下文。
  • Intervention Generator + Causal Evaluator：基于模板库生成候选方案，并用BN do-calculus 估算峰值/能耗收益。
  • Slate Bandit + Enhanced Feedback Module：以多目标奖励挑选Top-K方案，结合用户喜好/拒绝原因闭环自适应。

🔄 完整信息流

上下文输入 → 方案生成（候选集合）→ 因果干预评估（BN do-calculus）→ Bandit排序与Top-K推荐 → 用户反馈（选择/拒绝原因/点赞/执行确认）→ 在线学习更新（Bandit策略 + 偏好模型 + 生成策略演化）
上下文编码

多维输入（季节、建筑属性、实时负荷）经由上下文编码器生成状态摘要，驱动下游规划。

因果+Bandit融合

BN推断评估干预影响，Slate Bandit依据多目标奖励函数选择Top-K方案并持续在线更新。

反馈闭环

采集用户喜好、拒绝原因与执行确认，触发动作用屏蔽与策略自适应，确保安全落地。
