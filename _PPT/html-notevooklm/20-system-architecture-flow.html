폐루프 실행 및 피드백 경로

🧩 6대 모듈 협업 (텍스트 개요)

  • Context Encoder: 계절, 건물 형태, 실시간 부하 및 모델 내부 상태를 추출하여 견고한 컨텍스트 생성.
  • Intervention Generator + Causal Evaluator: 템플릿 라이브러리 기반 후보 방안 생성 및 BN do-calculus로 피크/에너지 효율 이득 추정.
  • Slate Bandit + Enhanced Feedback Module: 다목적 보상으로 Top-K 방안 선택, 사용자 선호/거부 원인 결합하여 폐루프 적응.

🔄 전체 정보 흐름

컨텍스트 입력 → 방안 생성 (후보 집합) → 인과 개입 평가 (BN do-calculus) → Bandit 정렬 및 Top-K 추천 → 사용자 피드백 (선택/거부 원인/좋아요/실행 확인) → 온라인 학습 업데이트 (Bandit 전략 + 선호도 모델 + 생성 전략 진화)
컨텍스트 인코딩

다차원 입력(계절, 건물 속성, 실시간 부하)이 컨텍스트 인코더를 통해 상태 요약을 생성하여 다운스트림 계획을 주도.

인과 + Bandit 융합

BN 추론으로 개입 영향 평가, Slate Bandit이 다목적 보상 함수에 따라 Top-K 방안을 선택하고 지속적으로 온라인 업데이트.

피드백 폐루프

사용자 선호도, 거부 원인 및 실행 확인을 수집하여 동작 마스킹 및 전략 적응을 트리거하고 안전한 적용 보장.
